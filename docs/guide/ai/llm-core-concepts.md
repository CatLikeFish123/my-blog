# LLM 核心概念

大模型（LLM）的第一梯队核心概念。我们不谈复杂的数学推导，直接用最通俗、直观的语言把这些概念的本质、核心作用和它们之间的关系讲清楚。

## 核心架构与基石

### 1. Transformer

**大白话**：现代大模型的「发动机架构」。

**核心本质**：抛弃了以往逐字阅读的传统（如 RNN），实现了全文本并行处理。它就像一个高速运转的特征提取工厂。

**为什么重要**：支持超大规模数据并行训练，是所有现代 LLM（GPT, Claude, Llama）的底层骨架。

### 2. Attention（注意力机制）

**大白话**：让模型知道一句话里「谁和谁更相关」。

**核心本质**：计算文本中每个词对其他词的权重。例如在「苹果很好吃」和「苹果发布了新手机」中，两个「苹果」会因为注意力机制，分别与「好吃」或「手机」建立更强的联系，从而精准理解语境。

**为什么重要**：打破了长文本的记忆限制，让模型有了「抓重点」的能力。

### 3. Embedding（嵌入/向量化）

**大白话**：把文字翻译成计算机能懂的「高维数字坐标」。

**核心本质**：将任何文本（词、句、篇）转化为一串固定长度的数字（向量）。含义相近的词，在多维空间中的几何距离就很近（例如「猫」和「狗」的距离，比「猫」和「汽车」近）。

**为什么重要**：是模型理解语义、计算相似度的基础。

### 4. Token（标示/词元）

**大白话**：大模型吃进去、吐出来的「字词碎片」。

**核心本质**：文本处理的最小单位。它不一定是完整的单词或汉字，往往是片段（比如英文里的前缀/后缀，中文里的单字或常用词）。

**为什么重要**：大模型的计费（按 Token 收费）和算力消耗都以此为基本单位。

### 5. Context（上下文）

**大白话**：大模型写字时的「草稿纸和记忆空间」。

**核心本质**：模型在处理当前任务时，能够同时看到并参考的前文所有信息。

**为什么重要**：上下文窗口（如 128k, 1M）越大，模型一次性读长篇小说或写大项目代码的能力就越强。

## 性能与效率优化

### 6. KV Cache（键值缓存）

**大白话**：大模型生成文本时的「缓存备忘录」。

**核心本质**：在逐字生成文本时，模型需要重复计算前面所有字之间的注意力。KV Cache 把已经计算过的历史字词的注意力特征（Key 和 Value）存起来，下次直接用，不用重新计算。

**为什么重要**：空间换时间。极大地提升了大模型推理（生成速度）的效率，降低了延迟。

## 训练、微调与对齐

### 7. SFT（监督微调）

**大白话**：让模型从「乱说胡话的通才」变成「懂规矩的答题手」。

**核心本质**：在海量预训练后，用高质量的「输入-输出」对（Prompt-Response）来规范模型的回答格式。教它如何扮演客服、如何写代码、如何礼貌回答。

**为什么重要**：这是大模型真正能听懂人话、按指令办事的关键一步。

### 8. RLHF（基于人类反馈的强化学习）

**大白话**：给大模型配一个「人类裁判」，给它的回答打分。

**核心本质**：通过人类的偏好（哪个回答更好、更安全、更诚实）训练一个奖励模型，再用这个奖励模型去引导、纠正 LLM 的输出。

**为什么重要**：对齐（Alignment）的核心技术。确保模型不仅聪明，而且符合人类价值观，不输出有害或冒犯性内容。

### 9. LoRA（低秩适应微调）

**大白话**：给大模型做微调时，「不拆承重墙，只贴外墙砖」。

**核心本质**：微调时冻结大模型原本的数百亿参数（不改动），只在旁边外挂一个极小规模的低秩矩阵参数进行训练，最后把结果加回原模型。

**为什么重要**：穷人的微调神器。把显存消耗和算力成本降低了几个数量级，让消费级显卡也能微调大模型。

### 10. MoE（混合专家模型）

**大白话**：把一个超级大模型拆成一堆「专业小团队」，外加一个「调度员」。

**核心本质**：模型内部包含多个专家网络（Experts），每次输入一个 Token 时，门控路由（Router）只激活其中 1-2 个最擅长该领域的专家来计算。

**为什么重要**：用更低的计算成本换取更大的模型容量（如 GPT-4, Mixtral）。虽然参数总量极大，但每次推理只消耗一小部分算力。

## 应用生态与工具调用

### 11. Function Calling（函数调用）

**大白话**：大模型自己不会算数和查天气，但它会给人发指令、调工具。

**核心本质**：模型根据用户的提问，判断是否需要调用外部工具，并自动输出标准的结构化参数（通常是 JSON）。比如用户问「明天北京天气」，模型输出 `{"tool": "weather", "city": "Beijing", "date": "tomorrow"}`，由代码执行后把结果喂回给模型。

**为什么重要**：大模型连接现实世界的桥梁，让它从「只能聊天」变成「能办实事」。

### 12. MCP（Model Context Protocol / 模型上下文协议）

**大白话**：AI 界的「USB 统一接口标准」。

**核心本质**：由 Anthropic 提出的开放标准。过去每个 AI 工具（查数据库、读本地文件）都要单独为大模型写接口。MCP 提供了一个统一协议，让任何数据源和工具都能即插即用地连接到任何 LLM。

**为什么重要**：极大简化了开发流程，构建了统一的 AI 生态连接标准。

### 13. Agent（智能体）

**大白话**：把大模型作为大脑，配上眼睛、手脚和记忆的「全自动 AI 打工人」。

**核心本质**：一个具备规划（Planning）、记忆（Memory）、工具使用（Tools）和执行（Action）能力的自主系统。你给它一个模糊的目标（如「帮我写一份 2026 年新能源汽车行业报告并做成 PPT」），它能自己拆解步骤、查资料、反复修改并最终完成。

**为什么重要**：大模型应用的终极形态，从「一问一答」走向「自主完成复杂任务」。
